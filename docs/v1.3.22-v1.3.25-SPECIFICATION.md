## v1.3.22: overrideNoiseGate / overrideVolumeMultiplier 追加

### 背景
- iPhoneの準備ページとトレーニングページで異なる音量設定が必要
- 準備ページ: BGMなし → noiseGate 25%でノイズブロック
- トレーニングページ: BGMあり（ダッキング発生）→ noiseGate 15%で声を通す

### 問題
DeviceDetection.tsの設定は全ページ共通のため、ページごとに設定を変えられなかった。

### 解決策
`updateSelectors()`で`noiseGate`と`volumeMultiplier`を上書きできる機能を追加。

### 変更ファイル
- `src/components/AudioDetectionComponent.ts`
  - `AudioDetectionConfig`に`overrideNoiseGate?: number`追加
  - `AudioDetectionConfig`に`overrideVolumeMultiplier?: number`追加
  - `updateSelectors()`でoverride値を受け取れるように型定義を拡張
  - `_getProcessedResult()`でoverride値を優先して使用

### 使用例
```javascript
// iPhoneのダッキング対策
await audioDetector.updateSelectors({
    volumeBarSelector: '#training-volume-progress',
    overrideNoiseGate: 0.15,       // 25%→15%に下げる
    overrideVolumeMultiplier: 2.5  // 音量表示を上げる
});
```

### ログ出力
```
🚪 [v1.3.22] overrideNoiseGate set to: 0.15 (15.0%)
🔊 [v1.3.22] overrideVolumeMultiplier set to: 2.5
🔍 [_getProcessedResult] volumeAsPercent:25.03% noiseGate:15.00% (OVERRIDE) deviceSpecs:OK
```

## v1.3.23: Android対応とデバイス別override制御

### 背景
- Androidが「PC」として扱われていた
- iPhoneのダッキング対策用overrideが他デバイスに悪影響を与える可能性

### 変更内容

#### 1. Android検出ロジック追加
```typescript
// DeviceDetection.ts
const isAndroid = /Android/i.test(userAgent);

if (isAndroid) {
  deviceType = 'Android';
}
```

#### 2. DeviceSpecs型にAndroid追加
```typescript
// types/index.ts
deviceType: 'iPhone' | 'iPad' | 'Android' | 'PC';
```

#### 3. Android専用最適化設定
```typescript
case 'Android':
  return {
    sensitivity: 2.0,
    noiseGate: 0.08,        // 8%
    volumeMultiplier: 2.5,
    smoothingFactor: 0.1
  };
```

## v1.3.24: iPad volumeMultiplier調整

### 背景
iPad準備ページで音量バーがすぐに100%に達する問題。

### 問題分析
ログ分析: `volumeAsPercent 36-65%` × `volumeMultiplier 3.0` = **100%超え**

### 解決策
iPad volumeMultiplier: **3.0 → 1.2** に削減

---

## v1.3.25: iPad noiseGate調整・iPadダッキング対策

### 背景
1. iPad準備ページで常時ノイズを拾う問題
2. iPadでもダッキングが発生することが判明

### 問題分析
ログ分析: 静寂時でも `volumeAsPercent 8%〜10%` のノイズ発生
noiseGate 5%では不十分でノイズがスルー

### 解決策

#### 1. DeviceDetection.ts - iPad noiseGate調整
```typescript
case 'iPad':
  return {
    sensitivity: 2.5,
    noiseGate: 0.12,         // 5% → 12% (ノイズブロック)
    volumeMultiplier: 1.2,   // v1.3.24で調整済み
    smoothingFactor: 0.1
  };
```

#### 2. trainingController.js v4.12.2 - iPad override追加
```javascript
const isIPhone = window.DeviceDetector?.isIPhone?.() || /iPhone/.test(navigator.userAgent);
const isIPad = window.DeviceDetector?.isIPad?.() || /iPad/.test(navigator.userAgent) ||
               (/Macintosh/.test(navigator.userAgent) && 'ontouchend' in document);

if (isIPhone) {
    selectorConfig.overrideNoiseGate = 0.15;
    selectorConfig.overrideVolumeMultiplier = 2.5;
} else if (isIPad) {
    selectorConfig.overrideNoiseGate = 0.08;     // ダッキング対策
    selectorConfig.overrideVolumeMultiplier = 4.0; // v4.12.4で調整
}
```

## 既知の問題: iOS Safari getUserMediaバグ

### 概要
WebKit Bug 230902: ページリロード後に`getUserMedia()`を再呼び出しすると、マイク入力レベルが大幅に低下する。

### 症状
- リロード前: `volumeAsPercent 60-75%`
- リロード後: `volumeAsPercent 15-20%`（約4分の1に低下）

### 対策
- **アプリ側対策**: preparation-pitchpro-cycle.js v4.12.5でiOS/iPadOSリロード時に警告表示
- **根本解決**: Appleのバグ修正待ち

### 参照
- WebKit Bug: https://bugs.webkit.org/show_bug.cgi?id=230902
